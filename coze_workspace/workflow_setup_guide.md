# 治愈系漫画工作流 - coze 手动搭建指南

> 这个工作流是参考原文教程结构,简化版。**用户按步骤在 coze 界面手动配置即可**(无需编程经验,每个节点 1-2 分钟)。

## 0. 前置准备

1. 登录 https://code.coze.cn (扣子编程)
2. 确认账户有余额(image2 是付费模型,跑一次约 0.5-2 元)
3. 进入"资源库" → 右上角"+" → "创建工作流"
4. 命名:`healing_comic_workflow`
5. 描述:`输入主题,生成 4 张治愈系漫画`

## 1. 开始节点(Start)

| 字段 | 值 |
|------|-----|
| 节点类型 | 开始 |
| 输入参数 1 | `title`(string,必填) — 漫画主题,如"下班后" |
| 输入参数 2 | `key`(string,选填) — image2 API key(可留空,用账户默认) |

**输出**:`title`, `key`

## 2. 节点 1: 生成治愈文案(LLM)

| 字段 | 值 |
|------|-----|
| 节点类型 | 大模型 |
| 模型 | 豆包 / DeepSeek(任选) |
| 系统提示词 | 复制 `prompts.md` 里的"提示词 1:治愈文案生成器" |
| 用户提示词 | `主题:{{title}}\n请围绕这个主题,创作一组 4 段治愈系短文案,严格按 JSON 数组输出。` |
| 输出变量 | `texts`(string) — JSON 数组字符串 |

**连接**:开始节点 → 节点 1

## 3. 节点 2: 格式转化(Plugin)

| 字段 | 值 |
|------|-----|
| 节点类型 | 插件 |
| 插件 | 剪映小助手 → str_to_list |
| 输入 | `{{节点1.texts}}` |
| 参数配置 | `delimiter: "\n"`,`output_format: "array"` |
| 输出 | `text_list`(array of string) |

**连接**:节点 1 → 节点 2

## 4. 节点 3: 生成参考图(Plugin - image2)

| 字段 | 值 |
|------|-----|
| 节点类型 | 插件 |
| 插件 | GT2_图片生成 → image2 |
| prompt | 复制 `prompts.md` 里的"提示词 3" |
| 输出 | `ref_image`(string - URL) |

**连接**:开始节点 → 节点 3(用 title 决定主色调/风格)

## 5. 节点 4: 批处理(Batch)

| 字段 | 值 |
|------|-----|
| 节点类型 | 批处理 |
| 输入数组 | `{{节点2.text_list}}` — 遍历每条文案 |
| 批处理体输入 | `item`(单条文案), `ref_image`(参考图,从节点 3 传入) |
| 批处理体输出 | `image_url`(string) |

**连接**:节点 2 → 节点 4(批处理);节点 3 → 节点 4(传 ref_image)

### 批处理体: 节点 4.1 - 生成图片提示词(LLM)

| 字段 | 值 |
|------|-----|
| 节点类型 | 大模型 |
| 模型 | 豆包 / DeepSeek |
| 系统提示词 | 复制 `prompts.md` 里的"提示词 2:漫画图片提示词生成器" |
| 用户提示词 | `请将以下文案转化为英文图片 prompt:\n{{item}}` |
| 输出 | `image_prompt`(string) |

### 批处理体: 节点 4.2 - 生成漫画图片(Plugin - image2)

| 字段 | 值 |
|------|-----|
| 节点类型 | 插件 |
| 插件 | GT2_图片生成 → image2 |
| prompt | `{{节点4.1.image_prompt}}` |
| reference_image | `{{ref_image}}` |
| size | 1024x1024 |
| num | 1 |
| 输出 | `image_url`(string) |

**连接**:节点 4.1 → 节点 4.2;批处理入参 → 节点 4.1, 节点 4.1 → 节点 4.2

## 6. 结束节点(End)

| 字段 | 值 |
|------|-----|
| 节点类型 | 结束 |
| 输出参数 1 | `images`(array) — `{{节点4.image_url}}` |
| 输出参数 2 | `count`(number) — `{{节点4.image_url.length}}` |

**连接**:节点 4 → 结束节点

## 7. 试运行

1. 点击右上角"试运行"
2. 输入 `title` = "下班后"
3. 等待 1-3 分钟
4. 查看结束节点的输出: 应该有 4 张图片 URL

## 8. 发布(可选)

- 工作流保存后,在扣子机器人或智能体里可以调用这个工作流
- 或导出为 API,集成到自己的应用

## 常见问题

**Q1: image2 跑失败?**
A: 余额不足。充值或在插件里换其他免费模型(质量会差)。

**Q2: 输出文案不是 JSON 格式?**
A: 在系统提示词里强调"严格 JSON",或者把 LLM 模型换成更稳定的(豆包 1.5 Pro / DeepSeek V3)。

**Q3: 参考图风格不一致?**
A: 在 image2 参数里加 `reference_strength: 0.6`(可调,0.4-0.7 常用)。

**Q4: 想增加漫画数量?**
A: 改 `prompts.md` 里的"输出 4 段"为"输出 6 段",批处理自动跑 6 张。
